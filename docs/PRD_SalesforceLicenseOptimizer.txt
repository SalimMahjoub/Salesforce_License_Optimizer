Product Requirements Document (PRD)
Salesforce License Optimizer
Version: 1.0
Date: 29 décembre 2025
Auteur: Architecture Team
Statut: Draft for Review

1. Executive Summary
Salesforce License Optimizer est un SaaS d'optimisation financière qui analyse automatiquement l'utilisation réelle des licences Salesforce pour identifier 20-60% de gaspillage typique et générer un plan d'optimisation sur-mesure.
Problème métier

Une licence Sales Cloud coûte ~150€/mois/utilisateur
Une entreprise de 500 employés paie 600k-1,2M€/an en licences Salesforce
40-60% de ces licences sont mal utilisées ou inutiles
Aucune visibilité pour le DSI, CFO ou Salesforce lui-même
L'audit manuel est chronophage et techniquement complexe

Proposition de valeur

ROI immédiat: 5k-20k€ d'économies détectées dès le premier scan
Zéro risque sécurité: Analyse uniquement métadonnées et logs (zero-knowledge)
Setup rapide: 3h de travail humain total par client
Monitoring continu: Rétention via alertes sécurité et conformité


2. Objectifs Produit
Objectifs Business (6-12 mois)

Générer 5-20k€ d'économies mensuelles par client en moyenne

4. Functional Requirements
4.1 Core Features (MVP)
FR-001: OAuth Connection to Salesforce
Priority: P0 (Critical)
User Story: En tant que DSI, je veux connecter mon org Salesforce de manière sécurisée pour autoriser l'analyse.
Acceptance Criteria:

OAuth 2.0 flow standard Salesforce
Demande uniquement les permissions minimales requises:

api (lecture métadonnées)
refresh_token (connexion persistante)


Pas d'accès aux données business (opportunités, contacts, comptes)
Stockage sécurisé des tokens (chiffrement at-rest)
Révocation possible côté client à tout moment
Temps de setup: <30 minutes

Technical Notes:

Utiliser Salesforce Connected App
Scopes: api, refresh_token, offline_access
Exclure explicitement: full, custom_permissions


FR-002: Data Collection Engine
Priority: P0 (Critical)
User Story: En tant que système, je collecte automatiquement les métadonnées et logs d'utilisation sur 30 jours.
Acceptance Criteria:

Collecte quotidienne automatique via Salesforce APIs:

UserLogin (dernière connexion, fréquence)
Profile et PermissionSet (droits attribués)
SetupAuditTrail (activités admin)
EventLogFile (field usage, report views, dashboard views)
Metadata API (objets custom, champs custom utilisés)


Période de collecte: minimum 30 jours (configurable)
Agrégation par utilisateur + profil + licence type
Stockage format structuré (JSON/Parquet)
Zéro accès aux données sensibles (contacts, opportunités, etc.)

#### FR-003: User Classification Algorithm
**Priority:** P0 (Critical)  
**User Story:** En tant que système, je classifie automatiquement chaque utilisateur dans 5 catégories d'utilisation.

**Classification Logic:**

| Catégorie | Utilisation | Critères | Recommandation Typique |
|-----------|-------------|----------|------------------------|
| **Zombie** | 0-10% | Pas de connexion depuis 90+ jours OU <2 connexions/mois | Désactiver licence |
| **Ghost** | 10-30% | Connexion occasionnelle mais <5 actions/mois | Downgrade → Platform ou Chatter (25€/mois) |
| **Casual** | 30-60% | Utilise features basiques uniquement (ex: lecture seule) | Downgrade → Sales Essentials (75€/mois) |
| **Power User** | 60-90% | Utilise majorité des features disponibles | Licence actuelle OK |
| **Super User** | 90-100% | Utilise toutes les features + besoins avancés | Considérer upgrade (Einstein, etc.) |

**Scoring Formula (exemple simplifié):**
```
Utilization_Score = (
    login_frequency_score * 0.3 +
    features_used_score * 0.4 +
    data_creation_score * 0.2 +
    advanced_features_score * 0.1
) * 100
```

**Acceptance Criteria:**
- Classification mise à jour quotidiennement
- Historique conservé (évolution dans le temps)
- Règles de scoring configurables par admin
- Export CSV de la classification

#### FR-004: Optimization Plan Generation
**Priority:** P0 (Critical)  
**User Story:** En tant que CFO, je reçois un plan d'optimisation chiffré avec économies potentielles par action.

**Plan Components:**
1. **Executive Summary:**
   - Coût actuel total mensuel/annuel
   - Économies potentielles totales
   - Top 3 quick wins

2. **Recommendations détaillées:**
   - Liste par utilisateur avec:
     - Nom/Email
     - Licence actuelle (coût mensuel)
     - Licence recommandée (coût mensuel)
     - Économie mensuelle/annuelle
     - Justification (classification + métriques)
     - Niveau de risque (Low/Medium/High)

3. **Security Alerts:**
   - Utilisateurs avec permissions excessives
   - Profils avec "Modify All Data" non utilisé
   - Accès admin inutiles

**Acceptance Criteria:**
- Génération automatique après 30 jours de collecte
- Format PDF + dashboard interactif
- Calculs certifiés (prix Salesforce officiels)
- Filtrage par département/entité/coût minimum
- Simulation "What-if" (impact si j'applique ces 10 recommandations)

Logs système conservés 1 an

7. Technical Constraints
TC-001: Salesforce API Limitations

API Call Limits: 15,000 calls/24h (Enterprise) ou 1,000,000 calls/24h (Unlimited)
Bulk API: Max 10,000 records par batch
EventLogFile: Disponible uniquement avec certaines licences (vérifier éligibilité)
Mitigation: Implémenter rate limiting + caching + collecte incrémentale

TC-002: Data Volume

Orgs testées: 100-10,000 utilisateurs
Volumétrie: ~1-5MB de métadonnées par org par jour
Rétention: 90 jours de données historiques
Storage estimé: ~100GB pour 500 orgs sur 90 jours

TC-003: Salesforce Editions Support

Supporté: Enterprise, Unlimited, Performance
Non supporté: Essentials, Professional (APIs limitées)
Validation: Détecter édition lors onboarding et afficher compatibilité

9. Pricing Strategy (Indicative)

€2/utilisateur Salesforce/mois
Exemple: 500 users = €1,000/mois = €12k/an
